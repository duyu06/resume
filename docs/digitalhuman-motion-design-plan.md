# AI 客服数字人前端与 GSAP 动画改造计划

## 设计参考

从 MotionSites 的公开设计方向中抽取以下机制：

- AI Workflow Hero：把核心工作流放入首屏视觉叙事。
- Layered Depth：数字人形象、会话、工具调用和指标形成前后景层次。
- Interactive Discovery：鼠标/触摸移动时揭示客户画像、工具轨迹和风险信息。
- Prisma Creative Studio：编辑式大标题、细网格、卡片分组和克制的滚动入场。
- Dashboard UI：使用明确的导航、指标、工具日志和会话工作区。

不复制受限模板代码，只复用公开的布局与交互设计原则。

## 页面结构

1. 左侧应用导航：实时对话、订单、物流、库存、退款、会话记录和数据看板。
2. 数字人舞台：数字人形象、语音波形、在线状态和当前会话摘要。
3. 主会话区：客户与 Agent 对话、订单卡、快捷工具和输入框。
4. 右侧洞察区：客户档案、Agent 指标、工具轨迹和风险状态。
5. 滚动叙事区：展示 Agent 如何从意图识别、工具选择、结果验证到人工接管。

## GSAP 动画

- 首屏进入：导航、数字人、聊天区和洞察卡分阶段入场。
- 数字人呼吸：光环、波形和状态灯使用低频循环动画。
- 会话动画：新消息、订单卡和工具调用沿时间线出现。
- ScrollTrigger：滚动到工作流章节时依次高亮意图、工具、业务结果和人工接管。
- Layered parallax：数字人、背景网格和光晕使用不同速率移动。
- reduced-motion：关闭视差和循环动画，只保留状态变化。

## 保留功能

- 本地演示模式和真实 FastAPI 模式。
- 订单查询、库存、退款确认、情绪识别、人工转接和性能报告。
- 服务端 API Key 不进入公开前端。
- 桌面端和手机端完整可操作。

## 验收

- 1440×900 与 390×844 双端布局。
- 无横向溢出。
- 消息、工具调用、退款确认和人工转接均可运行。
- GSAP 不阻塞原有状态机。
- prefers-reduced-motion 下页面可完整操作。
- Chromium 自动验证 GSAP/ScrollTrigger 加载、四阶段滚动激活和新 SVG 数字人资产。
