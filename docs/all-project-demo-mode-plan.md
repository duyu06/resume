# 全项目演示模式实施与验收计划

## 目标

为作品集内所有可访问项目提供无需真实后端、API Key、GPU、RTC 服务或本地执行器即可运行的演示模式，并通过桌面端和手机端浏览器自动化验证。

演示模式必须明确标注为模拟数据，不能将模拟结果表述为真实生产服务结果。

## 覆盖项目

1. 果漾 AI 多模态内容生产平台
2. YOLA 跨境电商独立站
3. PrismPix AI 电商主图生成器
4. talk-to-fengge-live 数字人
5. WebRPA 自动化系统
6. Open WebUI 多模型本地化平台
7. 叫魂者多 Agent 叙事系统

## 统一要求

每个项目页面必须具备：

- 明确可见的“演示模式”标识
- “运行完整演示”入口
- “模拟异常”入口
- “重置演示”入口
- 步骤进度和操作日志
- 成功状态与失败状态
- 不依赖第三方服务即可完成的核心路径
- 手机端不遮挡原页面主操作
- `prefers-reduced-motion` 降级
- 演示数据与真实服务边界说明

## 项目演示路径

### 果漾 AI

`载入示例剧本 → 提取角色 → 生成四视图 → 生成分镜 → 生成镜头 → 合成成片`

异常路径：模拟模型超时，展示重试与任务恢复。

### YOLA

`进入系列 → 打开商品快速查看 → 选择商品 → 加入购物袋 → 更新购物袋数量`

异常路径：模拟库存重新校验失败。

### PrismPix

`载入示例商品 → 产品分析 → 生成 14 条 Prompt → 生成 14 个图片结果 → 分类筛选`

异常路径：模拟真实后端未配置或生成失败。

### 数字人

`选择角色 → 创建演示会话 → 模拟 RTC / WebSocket 状态 → 进入通话 → 查询状态 → 挂断`

异常路径：模拟 RTC 初始化失败。

### WebRPA

`载入默认工作流 → 运行节点 → 人工审批 → 命中断点 → 继续 → 完成`

异常路径：审批驳回或登录状态失效。

### Open WebUI

`选择模型 → 启动推理服务 → 输入 Prompt → 生成模拟回复 → 展示吞吐和显存状态`

异常路径：模拟显存不足并给出恢复建议。

### 叫魂者

`选择角色 → 输入行动 → 运行一轮 Agent 协作 → 更新世界状态 → 展示导演 Agent 结果`

异常路径：模拟角色越权读取信息并回滚当前回合。

## 自动化验收

### 桌面端

- Chromium 1440×900
- 所有 7 个 Demo 页面可加载
- 演示面板可打开和关闭
- 完整演示可运行到成功状态
- 异常演示可进入失败状态
- 重置后恢复初始状态
- 无未捕获 JavaScript 错误
- 无同源资源 404
- 无横向溢出

### 手机端

- Chromium 390×844
- 演示入口不遮挡页面核心按钮
- 面板可滚动且适配安全区
- 完整演示和异常演示均可操作
- 关闭后页面恢复滚动
- 无横向溢出

## 正式运行判定

### 已正式运行

- GitHub Pages 静态前端已部署
- 演示模式在正式线上 URL 可运行

### 未正式运行

- PrismPix Flask 后端未部署时，不进行真实图片生成
- talk-to-fengge-live Python / RTC / WebSocket 后端未部署时，不进行真实音视频通话
- WebRPA 本地 Windows 执行器未运行时，不控制真实网页或桌面
- Open WebUI / Ollama / vLLM 服务未部署时，不执行真实模型推理
- 果漾演示页不调用生产模型或生产数据

只有对应后端服务具备公开可访问地址、健康检查、密钥隔离和真实任务成功记录后，才能标记为“真实服务已运行”。
