# PrismPix、talk-to-fengge-live 与 WebRPA 接入验收计划

## 1. 验收目的

本计划用于确认以下三个接入项目在作品集中的可用性、交互完整性、双端适配和生产运行边界：

1. `/resume/demos/ai-ecommerce/` — PrismPix 兼容适配
2. `/resume/demos/digitalhuman/` — talk-to-fengge-live 兼容适配
3. `/resume/demos/rpa/` — WebRPA 兼容适配

验收必须区分三个层级：

- **L1：静态展示可用** — 页面、样式、导航和素材能正常加载。
- **L2：浏览器演示闭环可用** — 不依赖真实后端时，核心流程可以完整交互并产生状态反馈。
- **L3：真实业务服务可用** — 已连接正式后端、真实模型/RTC/执行器和持久化设施，可执行实际业务任务。

L1、L2 通过不代表 L3 已通过。

## 2. 当前运行状态判定

### 2.1 已具备

- 三个兼容前端已进入 `main` 分支。
- GitHub Pages 可托管三个页面及浏览器演示逻辑。
- 页面支持配置后端地址并切换真实后端标记。
- 浏览器演示模式不需要 API Key，可以完成主要交互流程。

### 2.2 尚未证明为正式生产运行

#### PrismPix

GitHub Pages 只能运行前端。真实图片生成还需要：

- PrismPix Flask/Python 后端长期在线；
- 模型或图像生成供应商 API Key 由后端安全管理；
- CORS、任务队列、文件存储和任务状态接口可用；
- 生成结果可持久化并接受质量验收。

#### talk-to-fengge-live

真实数字人通话还需要：

- Python 后端长期在线；
- Vidu/阿里云 RTC 配置有效；
- WebSocket 控制代理可连接；
- 摄像头、麦克风、音色克隆和会话结束接口通过真机验证；
- API Key、音频和角色数据符合安全及授权要求。

#### WebRPA

真实自动化执行还需要：

- Windows 本地或专用执行节点运行 WebRPA 服务；
- 目标浏览器/桌面/Android 环境可控制；
- 执行器心跳、账号隔离、任务队列、日志和失败恢复可用；
- 实际运行符合目标平台规则和账号权限。

因此，当前项目应表述为：

> **三个前端演示已上线并可交互；真实 PrismPix 生成、数字人 RTC 通话和 WebRPA 自动化执行尚不能认定为正式生产运行。**

## 5. 主作品集入口验收

### 5.1 流程

1. 打开作品集首页。
2. 滚动至“精选项目”。
3. 分别打开 AI 电商、数字人、RPA 项目详情。
4. 切换两张项目图。
5. 检查“查看交互 Demo”链接。
6. 手机端将弹窗内容滚动到底部。
7. 验证关闭按钮和 Demo 操作区仍处于视口安全区域。
8. 使用 Esc 关闭，确认主页面滚动恢复。

### 5.2 通过标准

- 项目名称与详情内容匹配。
- Demo URL 指向对应子页面。
- 图片切换有效。
- 手机端 CTA 不被底部 Dock 遮挡。
- 顶部关闭按钮不随正文滚走。
- 关闭后 `body` 不再保持 fixed/locked 状态。

## 6. PrismPix 验收

### 6.1 浏览器演示闭环

1. 上传测试产品图。
2. 修改 SKU、类目、视觉风格、平台、语言和附加要求。
3. 执行产品分析。
4. 检查分析结果是否包含输入字段。
5. 生成 14 条 Prompt。
6. 全部展开并修改第一条 Prompt。
7. 导出 Prompt JSON。
8. 校验文件名、条数和已修改内容。
9. 生成图片结果。
10. 校验完整模式为 14 个模块。
11. H 筛选显示 5 张，D 筛选显示 9 张。
12. 刷新页面，确认后端地址设置仍存在。

### 6.2 真实后端接口契约

使用 Mock Server 或测试后端验证：

- `GET /`
- `POST /api/generate`
- `GET /api/status/{task_id}`
- `POST /api/generate-prompts`
- `GET /api/prompts?ws=...`
- `POST /api/generate-images`
- `GET /api/image?ws=...&code=...`

### 6.3 正式运行验收门槛

除前端测试外，还必须通过：

- 使用真实产品图完成至少 3 个 SKU；
- 每个 SKU 成功生成 H1-H5 与 D1-D9；
- 失败任务可重试，浏览器刷新后任务可恢复查询；
- Logo、颜色、结构和包装文字通过人工一致性检查；
- API Key 未出现在前端源码、Network URL 或日志；
- 生成文件具有持久化存储和过期清理策略。

## 7. talk-to-fengge-live 验收

### 7.1 浏览器演示闭环

1. 创建新角色。
2. 设置角色名、头像地址、自定义音色、通话模式和人格。
3. 保存角色并刷新页面，确认角色仍存在。
4. 编辑角色名称。
5. 使用已授权参考音频地址执行音色克隆演示。
6. 保存并开始通话。
7. 验证五个连接步骤全部完成。
8. 验证 Live ID、RTC Channel、RTC User 已生成。
9. 切换麦克风状态。
10. 查询会话状态。
11. 验证计时器增长。
12. 挂断并确认状态为 `ended`。
13. 使用 Chromium Fake Media 或真机验证摄像头和麦克风检测。
14. 清空诊断日志。

### 7.2 安全验收

- API Key 只允许存入 `sessionStorage`，禁止进入 `localStorage`。
- 角色和人格内容在渲染时必须防止 HTML 注入。
- 音色克隆必须展示授权提醒。
- 页面不得把演示 Live ID 声称为真实 RTC 会话。

### 7.3 正式运行验收门槛

- 真实后端 `/api/config`、`/api/characters`、`/api/personas` 可用；
- 至少完成 10 次真实创建会话、RTC 连接和挂断；
- 通话成功率达到约定目标，连接超时和断线可恢复；
- Chrome Android 和桌面 Chrome 真机完成摄像头、麦克风测试；
- WebSocket 断线、API Key 无效、RTC 鉴权失败均有明确异常反馈；
- 音频、头像、角色和会话日志具有隐私与删除规则。

## 8. WebRPA 验收

### 8.1 工作流编辑

1. 搜索模块。
2. 使用分类筛选。
3. 点击添加节点。
4. 桌面端拖拽模块到画布。
5. 拖动已有节点改变位置。
6. 修改节点名称、参数 JSON、自动重试和断点。
7. 输入非法 JSON，确认配置不会被错误保存。
8. 缩放和适应画布。
9. 保存到本地并刷新验证。
10. 导出 JSON 并验证节点字段。
11. 新建空白工作流。
12. 导入 JSON 并验证节点恢复。

### 8.2 执行闭环

1. 运行默认流程。
2. 到达人工审批节点后暂停。
3. 审批通过。
4. 到达断点后点击继续。
5. 验证流程全部成功。
6. 再次运行并在审批节点驳回。
7. 验证失败计数和审计日志。
8. 执行单步，确认仅完成一个节点。
9. 运行过程中测试暂停、继续和停止。

### 8.3 正式运行验收门槛

- Windows 执行器心跳正常；
- 测试环境中真实打开网页、填写表单、采集数据和结果回写；
- 人工审批未通过前不得执行提交节点；
- 登录失效、元素变化、超时和断网可安全中止或重试；
- 不提供绕过平台风控、验证码或账号权限的能力；
- 每次任务保留任务 ID、账号环境、节点日志、截图和失败原因；
- 多账号环境彼此隔离，不发生 Cookie 或浏览器配置串用。

## 9. 缺陷分级

### P0 — 阻断上线

- 页面打不开或主要脚本报错；
- 核心流程无法开始或无法完成；
- 手机端主要按钮不可点击；
- 密钥泄露到前端或持久化位置错误；
- RPA 未审批即执行关键动作；
- 页面将演示结果伪装为真实业务结果。

### P1 — 必须修复

- 数据刷新后意外丢失；
- 状态、日志或导出内容错误；
- 移动端明显错位、遮挡或横向溢出；
- 真实后端异常没有用户反馈；
- 无法关闭弹窗或恢复页面滚动。

### P2 — 可排期优化

- 动画不够流畅但不阻断操作；
- 次要文案、间距或视觉一致性问题；
- 非核心设备或浏览器上的轻微样式差异。

## 10. 发布门禁

### 演示版上线门禁

必须同时满足：

- 本地生产构建成功；
- 桌面与手机共 8 条 E2E 场景全部通过；
- 无 P0、P1 缺陷；
- 截图、JSON 报告和 Markdown 报告上传至 Actions Artifact；
- 合并后在正式 GitHub Pages 再执行同一套 E2E；
- 正式站点测试全部通过后才标记“演示版已验收”。

### 正式生产运行门禁

必须额外满足：

- 三个真实后端/执行器分别部署在稳定环境；
- 域名、HTTPS、CORS、监控、日志、告警和备份配置完成；
- 真实 API Key 由服务端密钥系统管理；
- 完成真实数据和真机验证；
- 定义服务负责人、故障响应和回滚方案；
- PrismPix、数字人和 WebRPA 分别通过本计划中的 L3 验收门槛。

在此之前，项目对外描述应使用：

> “已上线的交互演示 / 兼容适配原型”

而不应使用：

> “正式生产系统已运行”
