003 — Data Governance

Raw dialogue.
Structured signal.

清洗、去重、脱敏、角色归属确认和低质量样本剔除，将原始记录整理为可训练、可追踪的数据集。

004 — Prompt Architecture

Context.
Memory.

使用角色设定、当前上下文、历史记忆与输出约束四层结构，让训练、在线调用与问题定位使用同一输入协议。

005 — Training Verification

Train.
Validate.

划分训练集与验证集，观察训练过程和异常样本表现，不把“任务完成”误当成“模型可用”。

006 — Evaluation

Measure.
Compare.

通过 26 条标准测试 Prompt 与 25 分制评分卡，对人物一致性、上下文、自然度、稳定性和格式合规进行统一评估。

01 / PRIVACY

数据授权与脱敏

不公开原始历史对话、训练权重或敏感业务数据；训练前需要确认授权范围并完成隐私处理。

02 / QUALITY

过拟合与机械复读

不能只观察训练是否结束，需要检查重复表达、角色僵化、事实混淆和异常样本表现。

03 / DECISION

Prompt 与微调边界

先使用 Prompt 验证方向；只有当长期角色稳定性与格式合规仍不足时，才评估微调投入。
